# 智能问数数据源与工作流交互参考进度

更新日期：2026-06-03

## 背景

本次实现目标是在现有 VAgent 前端平台内制作“智能问数底层工作流”的可交互参考，不连接真实数据库，不新增真实后端 API。核心范围包括数据源管理、角色权限配置、工作流“数据库查询”节点，以及可供研发预览的本地页面。

## 当前已完成

- 新增数据源管理路由与入口：`#/data-source`、左侧导航“数据 / 数据源管理”、`views/data-source.html`、`js/data-source.js`，并在 `index.html` 加载脚本。
- 实现前端数据源 store：`window.VAgentDataSourceStore`，包含数据源与查询日志本地存储、种子数据、密码掩码、模拟连接测试、SQL 安全校验、上游 SQL 变量解析和模拟执行输出。
- 数据源管理页已支持列表、空状态、新建、编辑、测试连接、启用 / 停用、删除影响提示，以及详情抽屉的基本信息、引用关系、查询日志。
- 数据源管理页已根据会议结论收敛：移除顶部状态统计卡片、移除列表“使用范围”列，新增“状态”列表达启用 / 关闭，“连接状态”表达最近测试结果。
- 数据源管理列表操作列已调整：只直显“测试连接”，编辑、启用 / 停用、删除收进三点更多菜单；操作列作为普通列随横向滚动条整体移动。
- 数据源管理列表容器已向页面底部延展，内容区独立承接剩余高度和横向滚动。
- 新建 / 编辑数据源弹窗已移除页面内权限设计、所属空间和 SSL；数据库类型与基础信息字段对齐，数据源描述与右侧 Username / Password 区域对齐。
- 数据源详情抽屉的“查询日志”已改为可筛选、可分页的列表表格形式，展示执行时间、工作流 / 节点、执行人、状态、行数、耗时、SQL 和错误信息；顶部去掉状态概览卡片，改为关键词搜索、执行状态筛选、每页条数选择；日志表只渲染当前页，前端模拟日志保留上限扩到 10000 条，适配后续成千上万条日志的展示需求。
- 角色管理已补充“数据源管理”功能权限，并按当前页面口径收敛为查询、新增、编辑、删除、测试连接；已移除启用 / 停用、查看表结构、同步表结构、查看查询日志等高级权限项。
- 工作流编辑器已新增节点库分类“数据处理 / 数据源”和“数据库查询”节点，拖入画布后可打开右侧配置抽屉。
- 数据库查询节点已对齐现有知识库节点风格：顶部标题可编辑，标题下方为描述输入，顶部保留测试运行、更多、关闭；保存收进更多菜单。
- 数据库查询节点字段说明已迁移到字段名旁圆圈感叹号 tooltip，tooltip 使用页面级浮层，避免被右侧抽屉截断。
- 数据库查询节点已按“输入 → 数据源 → 查询配置 → 失败重试 → 输出 → 运行结果”的闭环重排。
- 输入区已改为可编辑行式配置：支持新增、删除、编辑入参名、类型、来源变量和必填状态；没有输入时只保留标题和“+ 输入”。
- 查询配置已从“固定 SQL / 使用上游变量”双模式改为单一 SQL 输入框：既可直接写 SELECT SQL，也可插入输入参数变量或上游 SQL 变量；已去掉“基础格式化”按钮。
- 如果 SQL 输入框内容整体为 `{{llm1.output.sql}}` 这类 SQL 变量，测试执行会先解析变量，再执行 SELECT-only 等安全校验。
- 查询配置区已移除独立“安全校验”按钮；安全校验由测试运行、保存或执行前自动触发。
- “执行参数”已改为“失败重试”：去掉最大返回行数，保留查询超时时间，新增最大重试次数；返回字段信息、记录查询日志从节点配置中移除。
- 输出区已改为对齐阿里数据连接器的树形契约：`result Object` 下只展示 `content Array<Object>` 和 `isError Boolean`，不展示额外动态字段说明、执行提示、字段元信息、耗时、行数等调试字段。
- 运行结果默认不占位；点击顶部“测试运行”后展示状态、耗时、行数、执行 SQL、结果预览和错误信息。表格 / JSON 作为运行结果预览方式，不影响输出契约。
- 工作流编辑器画布已支持 `Delete` 删除节点、鼠标滚轮缩放、按住鼠标中键拖拽平移。
- 知识库管理与工作流知识库过滤节点已补齐异常联动：知识库停用会在工作流节点中显示异常；过滤字段判断改为知识库级严格匹配，不再回退全局默认字段；过滤字段被删除后保留原字段并提示重新配置；字段类型变化导致操作符不适用时保留原操作符并提示重新选择；知识库主配置页和过滤分组页都不展示顶部异常汇总，只保留具体知识库项与条件卡片内的异常提示；保存、测试运行、发布前都会拦截异常配置。
- 需求修订文档已同步到 `docs/smart_qa_frontend_interaction_requirement_revision.md`，明确研发应按“单 SQL 输入框”口径实现。

## 当前关键口径

- 数据库查询节点不再拆“固定 SQL”和“使用上游变量”两个模式。
- 输入区只定义业务入参，例如 `student_num`、`course_num`。
- 查询配置区只有一个 SQL 输入框：直接写 SQL、插入业务入参、插入整段上游 SQL 变量都在这里完成。
- 安全校验必须保留，但不作为手动按钮暴露。
- 失败重试区只暴露最大重试次数和查询超时时间，不暴露最大返回行数。
- 输出结构固定为 `result Object`，且只包含 `content` 和 `isError`；表格 / JSON、执行 SQL、耗时、行数、错误详情只属于测试结果预览。

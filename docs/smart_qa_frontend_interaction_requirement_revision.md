# 智能问数前端交互需求修订建议

更新日期：2026-06-03

## 修订背景

当前原型已从“按需求文档逐项堆配置”调整为更贴近 VAgent 现有工作流编辑器的节点抽屉形态。原需求中部分内容偏后端管理表单，与现有节点的顶部操作、输入输出声明、分段配置方式不一致，需要回写到需求文档，避免研发按旧口径实现后返工。

本修订只聚焦前端页面与交互，不涉及真实数据库连接、后端 API、真实权限校验。

## 总体原则

- 数据库查询节点遵循现有工作流节点抽屉风格，重点参考知识库节点的顶部结构、输入输出展示、分段间距和轻量操作。
- 页面内不堆大段说明文字。字段说明优先放入字段名旁的圆圈感叹号 tooltip，tooltip 使用页面级浮层，避免被右侧抽屉截断。
- 节点配置形成闭环：输入定义从上游接收什么，查询配置定义如何形成 SQL，失败重试控制异常重跑边界，输出声明给下游消费，运行结果只服务当前测试预览。
- 输出结构是给下游节点消费的稳定契约；表格 / JSON 只是运行结果的预览方式，不作为节点输出契约配置。

## 需求项修订

| 原需求项 | 修订建议 | 原因 |
|---|---|---|
| 配置区包含：基础配置、数据源选择、SQL 配置、查询控制与输出 | 改为：顶部标题与描述、输入、数据源、查询配置、失败重试、输出、运行结果 | “基础配置”里的节点名称不应在正文重复出现，标题应放在抽屉顶栏；“查询控制与输出”混在一起不利于理解 |
| 节点名称在基础配置内填写 | 改为：抽屉顶栏“数据库查询”可编辑，标题下方只保留描述输入 | 与知识库节点一致，减少正文表单负担 |
| 标题下方展示说明文案，例如“仅展示当前用户有权且已启用的数据源” | 改为：字段名旁圆圈感叹号 tooltip 展示说明 | 保持节点抽屉干净，避免小字堆叠影响扫描 |
| SQL 支持两种模式：固定 SQL、使用上游变量 | 改为：不拆两个模式。只保留一个 SQL 输入框，既可直接写 SELECT SQL，也可插入输入参数或上游 SQL 变量 | 对用户更通俗：最终执行的 SQL 就来自这一个框。避免“输入区 sql 行”和“查询配置 SQL 来源变量”重复表达 |
| 变量选择器提供固定示例变量 | 改为：SQL 输入框下方展示“插入变量”按钮，优先展示已配置输入参数，如 `{{student_num}}`；同时保留少量上游 SQL 示例变量，如 `{{llm1.output.sql}}` | 变量按钮应服务当前配置，不应看起来像写死示例 |
| 使用上游变量模式下单独选择 SQL 来源变量 | 改为：如果整段 SQL 来自上游，就在 SQL 输入框内插入该变量，例如 `{{llm1.output.sql}}`；测试执行时先解析变量，再做安全校验 | 保持“一个入口形成 SQL”的逻辑闭环 |
| 查询配置区展示“基础格式化”按钮 | 去掉该按钮，不提供自动格式化功能 | 原型阶段避免增加非核心 SQL 编辑能力，减少按钮干扰 |
| 查询控制默认包含最大返回行数、超时、输出格式、返回字段信息、查询日志 | 改为“失败重试”：保留查询超时时间，新增最大重试次数，去掉最大返回行数 | 当前节点配置重点是失败后的执行策略；最大返回行数不再作为页面配置项暴露 |
| 输出格式表格 / JSON 放在节点配置里 | 移到运行结果区，命名为“预览方式” | 表格 / JSON 只影响当前测试结果怎么看，不改变下游输出结构 |
| 返回字段信息开启，可配置 | 从页面配置中移除；字段信息只在运行结果预览中辅助展示，不作为输出变量暴露 | 字段信息偏调试和预览，不应污染下游输出契约 |
| 查询日志开启且 V1 不可关闭 | 从节点配置中移除；作为平台审计能力默认记录，在数据源详情“查询日志”里查看 | 不可关闭的开关没有交互价值，放在节点里增加困惑 |
| 输出变量用灰底卡片或多列平铺展示 | 改为对齐阿里数据连接器的输出结构：`result Object` 下只展示 `content Array<Object>`、`isError Boolean` | 输出变量只保留下游真正需要消费的工具执行结果，不暴露 SQL、字段元信息、耗时、行数等调试细节 |
| 运行结果区域默认展示空状态 | 改为：默认不展示运行结果；点击顶部“测试运行”后才展示 | 运行结果是测试动作产生的，不应在未执行时占据页面 |
| 模拟执行按钮放在底部固定操作栏 | 改为：顶部操作区保留“测试运行”图标按钮 | 与知识库节点顶部操作习惯一致 |
| 保存按钮顶部直显或底部固定 | 改为：保存收进顶部“三点更多”菜单 | 节点配置更像工作流编辑器内的轻量抽屉 |
| 查询配置区展示独立“安全校验”按钮 | 移除按钮；安全校验在测试运行、保存或执行前自动触发，失败时在查询配置下方显示错误提示 | 安全校验是执行前置规则，不应成为用户必须手点的额外动作 |
| 数据源管理列表操作全部直显 | 改为：只直显“测试连接”，编辑、启用/停用、删除收进三点更多菜单 | 避免表格操作列过宽、错位 |
| 数据源管理操作列固定在右侧 | 改为：操作列作为普通列，跟随底部横向滚动条整体移动 | 避免固定列造成视觉断层 |
| 工作流画布只支持固定视图 | 增加：Delete 键删除节点、鼠标滚轮缩放、按住鼠标中键拖拽平移 | 这是工作流编辑器的基础交互预期 |

## 数据库查询节点建议结构

1. 顶部：节点图标、可编辑标题、测试运行、更多、关闭。
2. 描述：标题下方一行描述输入，占位“请输入描述...”。
3. 输入：行式结构，左侧为入参名、必填标识、类型，右侧为上游变量来源；支持新增、删除、编辑。
4. 数据源：选择当前用户有权且已启用的数据源，提供“前往数据源管理”入口。
5. 查询配置：单一 SQL 输入框。可直接写 SELECT SQL，也可插入 `{{student_num}}`、`{{llm1.output.sql}}` 等变量；不再拆“固定 SQL / 使用上游变量”模式；不展示“基础格式化”按钮。
6. 失败重试：最大重试次数、查询超时时间。
7. 输出：树形输出契约，统一外层为 `result Object`，下方只包含 `content Array<Object>` 和 `isError Boolean`。
8. 运行结果：仅测试运行后出现，展示状态、耗时、行数、执行 SQL、前 20 行预览、错误信息，预览方式支持表格 / JSON。

## 7.5 输入字段修订建议

原“输入”中不应再放“节点名称 / 节点描述”。这两个字段已经迁移到节点抽屉顶部：节点名称为顶栏可编辑标题，节点描述为标题下方一行描述输入。

建议将 7.5 输入改为：

| 字段 | 类型 | 是否必填 | 说明 |
|---|---|---|---|
| 入参名称 | 输入框 | 是 | 定义当前节点从上游接收的参数名，例如 `student_num`、`course_num`。SQL 中可通过 `{{student_num}}` 引用。 |
| 参数类型 | 下拉选择 | 是 | 支持 `String`、`Number`、`Boolean`、`Object`、`Array<Object>`。默认 `String`。 |
| 是否必填 | 星号切换 | 否 | 标记该入参运行时是否必须有值。必填参数为空时，测试运行应给出错误提示。 |
| 值 / 来源变量 | 下拉选择 | 是 | 选择上游节点输出或开始节点变量，例如 `{{start.query}}`、`{{llm1.output.sql}}`。 |
| 操作 | 图标按钮 | 否 | 支持删除该输入行。 |

补充说明：

- 输入区只定义业务入参，不再因为“使用上游变量 SQL”额外生成一行固定 `sql` 输入。
- 查询配置区只有一个 SQL 输入框。SQL 可以直接写，也可以通过“插入变量”引用输入参数或上游 SQL。
- 没有输入时，不展示灰色空状态说明，只保留“输入”标题和“+ 输入”操作。

## 数据源管理页面建议结构

- 列表字段保留：数据源名称、数据库类型、状态、连接状态、创建人、创建时间、最近测试时间、操作。
- “状态”表达启用 / 关闭；“连接状态”表达最近连接测试结果，如已连接、连接失败、未测试。
- 列表移除“使用范围”列；权限范围如后续恢复，应放在详情中查看。
- 操作列：直显“测试连接”；“编辑 / 启用停用 / 删除”进入三点更多菜单。
- 操作列不固定，随横向滚动条一起移动。
- 新建 / 编辑弹窗移除页面内权限设计、所属空间、SSL；数据库类型与左侧基础信息字段对齐。

## 测试计划调整

- E2E 增加：数据库查询节点只展示一个 SQL 输入框，不展示“固定 SQL / 使用上游变量”切换。
- E2E 增加：查询配置区不展示“基础格式化”按钮。
- E2E 增加：SQL 输入框可插入输入参数变量，并能通过测试运行触发安全校验。
- E2E 增加：SQL 输入框内容为 `{{llm1.output.sql}}` 这类整段 SQL 变量时，测试运行先解析变量，再校验并执行模拟结果。
- E2E 增加：失败重试区展示最大重试次数、查询超时时间，不展示最大返回行数。
- E2E 增加：表格 / JSON 只影响运行结果预览，不改变输出结构。
- E2E 增加：数据源管理操作列“测试连接 + 三点更多菜单”，且操作列跟随横向滚动。
- E2E 增加：工作流画布 Delete 删除、滚轮缩放、中键拖拽平移。

## 不建议改动的原需求

- 本版只做前端交互参考，不真实连接数据库。
- 数据源先在数据源管理里统一创建，再在工作流节点中引用。
- SQL 安全校验必须保留：空 SQL、非 SELECT、写操作、多语句、上游变量空值都应拦截。
- 查询日志作为平台审计能力默认记录。
- 角色权限保留数据源管理核心权限项：查询、新增、编辑、删除、测试连接；暂不展示启用 / 停用、查看表结构、同步表结构、查看查询日志。

## 知识库过滤联动补充

- 知识库管理页停用知识库时，应写入共享知识库状态，并通知工作流编辑器刷新。
- 工作流知识库节点已选择的知识库如果被停用，应保留原选择，但显示“已停用 / 配置异常”，提示重新启用或更换知识库。
- 知识库字段被删除后，工作流过滤条件不应静默替换成其他字段，应保留原字段值；字段选择框显示“字段已删除：原字段名”，条件卡片显示异常提示，提示用户重新选择字段或删除该条件。
- 字段类型变化导致原操作符不再适配时，不应自动替换操作符；操作符选择框显示“操作符不适用：原操作符”，条件卡片显示异常提示，提示用户重新选择操作符。
- 存在字段已删除、操作符不适用、知识库停用等异常时，保存、测试运行、发布 / 执行前都应拦截，并给出明确错误；用户修复配置后才能继续。
- 新增选择知识库时，停用知识库不可选择；已选知识库后续停用时，只显示异常，不自动删除配置。

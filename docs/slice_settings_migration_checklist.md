# 切片设置迁移对比清单（QA 回归用）

## 入口与页面
- 迁移前：新建知识库流程 Step 3「索引设置」在创建向导弹窗内展示。
- 迁移后：文档列表工具栏「批量切片」进入独立页面区域「切片设置」展示 Step 3 的全部内容。

## 展示与布局
- 迁移前：Step 3 在创建向导的步骤容器内，左右分栏（设置 / 预览）。
- 迁移后：左右分栏布局保持一致（复用原 Step 3 DOM 结构、断点与间距类名不变）。

## 功能点核对（需 100% 等价）
### 切片策略
- 通用切片：选择、键盘操作（Enter/Space）、ARIA 状态（role="radio" / aria-checked）
- 模版切片：文件类型选择、增强策略展开/收起
- Excel 特殊提示：显示/隐藏逻辑

### 参数输入
- 分段标识符：输入框与下拉联动
- 分段最大长度：输入范围、边界值
- 切片重叠长度：输入范围、边界值

### 预处理与增强策略
- 空格/换行替换
- URL/邮箱删除
- 关联文件名
- 关联标题及子标题

### 预览与异常
- 切片预览区域渲染
- 预览文件选择
- 异常计数与上下导航
- 「查看原文」「重新切片」按钮

### 原文面板
- 原文面板显示/隐藏
- 放大/缩小/重置

## 交互状态核对
- hover / focus-visible / disabled / loading 的样式与行为
- 事件监听：不重复触发、不丢失

## 可访问性核对
- 关键交互元素可聚焦
- 快捷键：ESC 关闭设置页并返回文档列表
- 屏幕阅读器：标题、区域 role/aria 属性可被识别

## 回归步骤建议
1. 进入知识库详情页（文档列表）。
2. 点击「批量切片」进入「切片设置」。
3. 逐项验证上述功能点与交互状态。
4. 使用边界值（最小/最大/非法输入）触发校验与错误提示。
5. 验证键盘操作（Tab 导航、Enter/Space、ESC）。
6. Chrome / Firefox / Safari / Edge 全部通过后判定零阻塞缺陷。
